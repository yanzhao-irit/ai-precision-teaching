# 关于数据 · Data 说明（先读我）

> 给接手的同学：这个文件解释**为什么仓库里没有原始数据**、**数据到底在哪**、以及**怎么拿到测试数据并导入系统**。

---

## 1. 为什么 git 里看不到原始数据？

两个原因：

1. **隐私**：原始数据里有**真实学生姓名 + 学号**（属于个人信息），不能放进代码仓库（尤其不能放公开仓库）。所以 `.gitignore` 已经屏蔽了 `.xls / .xlsx / .doc / .docx / .zip`，这些文件**不会被提交**。
2. **数据库数据不在仓库里**：已经导入的数据存在 **Docker 数据卷**（`apt_pg_data`）里，也在 git 之外。

所以你 clone 下来后，**代码是完整的，但数据库是空的**——这是正常的，靠下面第 3 步把数据导进去。

---

## 2. 数据现在在哪？

| 数据 | 位置 | 说明 |
|---|---|---|
| 已导入的数据 | Docker 里的 PostgreSQL（卷 `apt_pg_data`） | 起好容器就在，关机重启也还在 |
| 原始文件（.xls/.xlsx/.zip） | **老师单独保管，不在仓库** | 需要测试时找老师要 |
| 你本地放的测试数据 | `data/raw/`（本目录下） | 这个目录已被 git 忽略，放进去不会误传 |

> 把要测试的原始文件放到 `data/raw/` 下即可，安全、不会进 git。

---

## 3. 怎么把数据导入系统？（不用敲命令，网页操作）

确保三件套都起好了（数据库 / 后端 / 前端，见根目录 `SETUP.md`），然后：

1. 浏览器打开 `http://localhost:5173`
2. 首页点 **「+ 新建课程」** → 填课程代码（如 `AI-BASE-2025`）和名称（如 `人工智能基础`）→ 创建并进入
3. 在课程里点 **「导入数据」**，把文件拖进对应区域：

| 上传区 | 文件类型 | 内容 | 示例文件名 |
|---|---|---|---|
| 课程数据 | `.xlsx` | 全班名单 + 综合成绩 + 学习进度（任务/讨论/章节访问） | `(2025-2026-1)-…_统计一键导出.xlsx` |
| 作业题库 | `.xls / .xlsx` | 题目、选项、正确答案、知识点 | `第1章…【单元测试】.xls` |
| 学生作业 | `.zip`（内含 `.doc`） | 每个 .doc = 一名学生的逐题作答 | 压缩包内：`学院-专业-班级-学号-姓名.doc` |

建议导入顺序：**作业题库 → 学生作业 → 课程数据**。

> 三类文件靠**学号**互相关联，所以同一门课要用同一批学生的数据。

---

## 4. 各类文件长什么样（字段速览）

- **作业题库 .xls**：一行一题，列含 `题目类型 / 大题题干 / 正确答案 / 答案解析 / 难易度 / 知识点 / 选项A…`。系统据此建题目、选项、知识点。
- **学生作业 .doc**（Word 2003 XML 格式）：正文是该生整份测验，逐题写着「学生答案 / 正确答案 / 题目得分」。系统据此得到**每生每题的对错**（错因诊断、掌握度全靠它）。
- **班级一键导出 .xlsx**：多个 sheet，主要用「学生学习进度详情」和「综合成绩」两个 sheet，得到名单、作业/综合成绩、任务完成数、讨论数、章节访问数等**聚合信号**（用于画像/预警，不含逐题对错）。

---

## 5. 注意事项

- **不要把 `data/raw/` 里的原始文件 commit 进 git**（`.gitignore` 已屏蔽，但别用 `git add -f` 强加）。
- 想清空数据库重来：`docker compose down -v` 再 `docker compose up -d`（会删掉所有已导入数据，重新建空表）。
- 数据处理的解析逻辑在 `data-pipeline/`（`etl/parsers.py` 解析三类文件，`run_etl.py` 是命令行入口；网页上传时后端会调用它）。想了解字段映射看那里。
