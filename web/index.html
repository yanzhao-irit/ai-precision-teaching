AI Teaching Diagnosis Prototype

First working prototype for explaining student mistakes with a knowledge graph.

1. Goal

This prototype helps understand why a student may have made a mistake in an AI course. Instead of only showing whether an answer is correct or incorrect, it links the mistake to course concepts and possible prerequisite gaps.

Example:

A student fails a question about Neural Network.
The system checks the knowledge graph and detects that the issue may come from Dot Product, Matrix, or Vector.

2. Knowledge graph logic

The course is represented as a graph. Questions are linked to the concepts they evaluate. Concepts are linked to the concepts required to understand them. 

Student → answered → Question
Question → evaluates → Concept
Concept → requires → Prerequisite Concept

This makes the diagnosis explainable because the system follows explicit links instead of only giving a black-box answer.

3. Example diagnosis

Student: S001
Failed question: Q005
Visible concept: Neural Network
Possible root cause: Dot Product
Recommendation: Review Dot Product before retrying Neural Network exercises.

This is the main value of the prototype: the visible error is not always the real cause. A mistake on a current topic may come from an older prerequisite.

4. Link with the four project problems

Problem 1 — Understanding mistakes:
The prototype gives a possible explanation, not only a score.
Problem 2 — Linking errors to concepts:
Each failed question is connected to a course concept and its prerequisites.
Problem 3 — Personalized support:
Each student gets a specific report with possible gaps and recommendations.
Problem 4 — Teacher support:
The class summary helps the teacher identify frequent difficulties.

5. Generated outputs

The diagnosis script generates:

  • Diagnosis CSV
  • Student report S001
  • Student report S002
  • Student report S003
  • Student report S004
  • Class summary

The student reports are easier to understand than raw Neo4j queries. They show the failed question, the visible concept, the possible root cause, an explanation and a recommendation.

6. How to run the prototype

Start Neo4j first, then run:

python scripts/run_demo.py

To open this web page:

python -m http.server 8000

Then open:

http://localhost:8000/web/
